      ESLint

      ESLint 是一个开源的 JavaScript 代码检查工具。

      npm 安装

      $ npm install --save-dev eslint                  # 本地安装
$ npm install --save-dev eslint-plugin-html      # 安装 .html 插件
$ npm install --save-dev eslint-plugin-vue       # 安装 .vue 插件，不能和 .html 插件同时使用。
$ npm install --save-dev babel-eslint            # 安装第三方解析器
$ npm install --save-dev eslint-config-standard
$ npm install --save-dev eslint-plugin-standard
$ npm install --save-dev eslint-plugin-import
$ npm install --save-dev eslint-plugin-node
$ npm install --save-dev eslint-plugin-promise   # 安装第三方规则

      安装说明

      • 不论是在编辑器中通过插件使用还是在命令行中运行命令使用，都需要安装 npm 包。
      • 如果全局安装 ESLint，必须全局安装插件和解析器。本地安装同理。
      • 如果是全局安装，可以使用编辑器插件自动检查文件，可以在项目目录下直接运行命令行选项。
      • 如果是本地安装，只有在运行项目时才会检查文件，必须进入 ./node_modules/.bin/ 目录下运行命令行选项。

      配置说明

      类型 配置说明 配置示例
      ESLint 配置
      1. 使用 JavaScript 注释把配置信息直接嵌入到一个代码源文件中。
      2. 使用命令行选项指定某个配置。
      3. 创建 .eslintrc.js 格式的配置文件，放在项目根目录中。
        另外还支持 .eslintrc.json 等格式。
      4. 在 package.json 文件里的 eslintConfig 字段中指定配置对象。
      1. 配置向导 eslint --init
      2. 本项目中使用的配置文件
      ESLint 规则

      所有的规则默认都是禁用的，可在配置文件中开启。

      1. 使用 "extends": "eslint:recommended" 字段启用推荐的规则。
      2. 使用 "extends": "standard" 字段启用第三方规则，需要安装 npm 包。
      3. 使用 "rules" 字段启用额外的规则。"off" 关闭规则，"warn" 开启警告，"error" 开启报错。
      ESLint 忽略
      1. 创建 .eslintignore 格式的配置文件，放在项目根目录中。
      2. 在 package.json 文件里的 eslintIgnore 字段中指定文件和目录。
      1. 本项目中使用的忽略文件
      检查 Vue
      1. 安装插件：eslint-plugin-vue。
      2. 启用插件：在 plugins 字段中添加 "vue"，关闭 "html"。
      3. 启用规则：在 extends 字段中添加 "plugin:vue/essential"。
      4. 启用解析器：将 "parser": "babel-eslint" 移入 parserOptions 字段中。因为 eslint-plugin-vue 插件使用 vue-eslint-parser 解析器，添加其他解析器会覆盖默认设置，导致插件不起作用。
      1. vue 项目中使用的配置文件

      使用方法

      平台 使用方法
      Sublime
      1. 安装插件 SublimeLinter。
      2. 安装插件 SublimeLinter-eslint。
      VS Code
      1. 安装插件 ESLint。
      2. 在用户设置中添加以下配置。
      命令行 
      $ eslint -h                     # 查看所有选项
$ eslint --init                 # 初始化向导，生成配置文件
$ eslint -c file.js             # 指定其它配置文件
$ eslint --ignore-path file.js  # 指定其它忽略配置文件
$ eslint file.js                # 检查文件
$ eslint --fix file.js          # 修复文件，但大部分是对空白的修复。

      3. 创建 .stylelintrc格式的配置文件，放在项目根目录中。
